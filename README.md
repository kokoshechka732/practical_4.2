# Лабораторная работа 4.2
Цель:
Научиться создавать простые графические интерфейсы с использованием библиотеки tkinter, организовывать обработку событий и интегрировать ранееразработанные программные модули в единое приложение. Закрепить понимание работы выбранной структуры данных через её визуальное представление.

Особенности реализации:
Все три библиотеки реализуют двустороннюю очередь с одинаковым набором операций: добавление и удаление элементов с обоих концов, получение размера, проверка на пустоту, преобразование в строку и очистка. Каждый элемент хранит целое число и текстовую строку из 9 символов.
В Python и C использовался односвязный список с указателями head и tail, что требует  O(n) для pop\_back, поскольку отсутствует ссылка на предыдущий. C++ управление памятью через malloc/free.
C++ STL реализация используется std::deque контейнер из стандартного набора, где все операции с обоих концов выполняются за O(1). 

Вывод:
На основе структуры данных «дек», разработанной на языке C в лабораторной работе 4.1, был выполнен перенос кода на C++ с сохранением логики работы односвязного списка. Полученный модуль был скомпилирован в виде динамической библиотеки и успешно интегрирован с графическим интерфейсом пользователя, реализованным на языке Python с использованием библиотеки tkinter, посредством модуля ctypes.
Параллельно была создана альтернативная реализация дека на чистом Python, также подключённая к GUI для сравнения производительности и удобства использования. Кроме того, был разработан третий модуль на C++ с применением готовых контейнеров и методов стандартной библиотеки STL, интеграция которого с Python осуществлялась через современный инструмент pybind11.
В результате выполнения данной работы были освоены следующие практические навыки: формирование динамических библиотек на C и C++, межъязыковое взаимодействие через ctypes и pybind11, использование контейнеров STL, а также разработка графических интерфейсов с помощью tkinter.
